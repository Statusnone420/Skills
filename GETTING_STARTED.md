# Getting started

This tutorial uses a disposable repository and keeps the first pass bounded.

1. Install the source skill as described in [INSTALL.md](INSTALL.md).
2. Start with `$docs map` to see the documentation topology and hot path.
3. Ask `$docs context the API migration` for sources, constraints, risks, and deliberately unloaded material.
4. Ask `$docs write an API migration guide`; verify the proposed claims before accepting edits.
5. Run `$docs check` and inspect missing links, anchors, duplicate titles, reachability, and the 16 KiB map/state budget.

For changes, `$docs update ...` revalidates affected evidence. Structural commands such as `init`, `migrate`, and `cleanup` preview first and require a later exact approval.
